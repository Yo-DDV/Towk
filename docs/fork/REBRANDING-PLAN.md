# Plan de rebranding

## Phase 1 — Inventaire

Recenser le nom Chatto dans le manifeste PWA, les icônes, le shell HTML, les textes, les URL de documentation, les labels OCI, la CLI, les métriques et les procédures. Distinguer les mentions d’attribution à conserver des éléments de marque remplaçables.

## Phase 2 — Décision

Valider le nom final, les domaines, l’identité visuelle, le propriétaire des nouveaux assets et la compatibilité avec les licences. Le domaine `towk.example` ne suffit pas à acter cette décision.

## Phase 3 — Migration bornée

Modifier d’abord les surfaces visibles et configurables, avec tests de manifeste, icônes maskables, service worker, titres, emails et liens source. Renommer les identifiants techniques persistants uniquement si le bénéfice justifie une migration et un rollback.

## Phase 4 — Vérification

Vérifier l’attribution, l’accès au code source, l’installation PWA, les upgrades, les liens profonds et la coexistence avec les données créées avant renommage.
