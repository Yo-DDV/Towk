# Code of conduct

Towk is committed to a welcoming, safe, and constructive community.

## Our standards

Examples of behavior that contributes to a positive environment include:

- showing empathy and respect for different experience levels and viewpoints;
- giving specific, evidence-based feedback about work rather than people;
- accepting responsibility, correcting mistakes, and sharing useful context;
- respecting privacy, security boundaries, and the time of other contributors.

Unacceptable behavior includes harassment, threats, discrimination, sexualized
attention, deliberate intimidation, doxxing, publishing private information,
or sustained disruption of project spaces.

## Enforcement

Project maintainers may edit, remove, reject, or lock contributions and may
temporarily or permanently restrict participation when behavior violates this
code. Enforcement should be proportionate, documented privately where
appropriate, and protective of reporters and affected people.

Report conduct concerns privately to the project owner through the contact
route listed on the maintainer's GitHub profile. Do not include sensitive
personal information in a public issue. Reports will be handled as
confidentially as practical; absolute confidentiality cannot be guaranteed when
action is required to protect someone.

This code applies in project repositories, discussions, issue trackers, review
threads, community spaces, and when someone officially represents Towk.
