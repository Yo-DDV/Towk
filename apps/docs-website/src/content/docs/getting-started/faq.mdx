Read [LICENSING.md](https://github.com/Yo-DDV/Towk/blob/main/LICENSING.md) for the
overview and [REUSE.toml](https://github.com/Yo-DDV/Towk/blob/main/REUSE.toml)
for the exact file boundary. If you modify AGPL-covered code and provide it to
users over a network, the AGPL generally requires you to offer those users the
corresponding source under the same license. This documentation is not legal
advice.

## Is Towk allowed to be an independent product?

Yes. The applicable open-source licenses allow the code to be used, modified,
redistributed, and supported subject to their conditions. Towk preserves
copyright notices, license texts, source availability, and factual attribution
to [Chatto](https://github.com/chattocorp/chatto). It uses a separate identity
and does not imply endorsement by ChattoCorp GmbH. The canonical Towk repository
is standalone and can still import compatible upstream changes while preserving
their provenance. See
[PROVENANCE.md](https://github.com/Yo-DDV/Towk/blob/main/PROVENANCE.md) and
[NOTICE](https://github.com/Yo-DDV/Towk/blob/main/NOTICE).

## Can I contribute?

Yes. Issues, discussions, and pull requests are public. Contributions are
accepted under the license already attached to the files they change. Read the
[contribution guide](https://github.com/Yo-DDV/Towk/blob/main/CONTRIBUTING.md)
before submitting code, documentation, generated output, or machine-assisted
work.

## Is Towk ready for production use?

Towk is pre-1.0 software. A pilot deployment is running, and the repository has
automated tests, dependency review, CodeQL, protected-branch checks, image
scanning, SBOM generation, and backups. That evidence does not guarantee that
every workload or environment is production-ready.

Pin deployments to an immutable revision, keep tested backups, monitor the
service, and validate upgrades in a staging environment. Public APIs may still
change during the `0.x` series.

## Does Towk collect telemetry or track users?

Towk does not include product analytics or third-party tracking, and a
self-hosted server does not send conversations or account data to the Towk
repository owner. Operators can expose local
[Prometheus metrics](/guides/operations/server-operations/#metrics) for their
own monitoring.

## Is Towk end-to-end encrypted?

Except for the media path used by voice and video calls, Towk is not generally
end-to-end encrypted. The server must decrypt messages before sending them to
authorized clients.

Towk encrypts message text and selected personal data at rest with per-user
keys. Attachments and some event metadata remain outside that application-level
encryption boundary and must be protected by the configured NATS or
S3-compatible storage. See
[Encryption at Rest and Data Erasure](/guides/operations/privacy-erasure/) for
the exact coverage and threat model.

## Does Towk federate data between servers?

No. Each Towk server powers one independent community. The client can connect
directly to multiple servers, but those servers do not copy or synchronize
community data with one another.

## Are there desktop and mobile apps?

The Progressive Web App is the current desktop and mobile client. Install it
from a supported browser or add it to the home screen. Dedicated app-store
packages are not currently published.

## Where can I get help or report a problem?

Use [GitHub Discussions](https://github.com/Yo-DDV/Towk/discussions) for setup
questions and [GitHub Issues](https://github.com/Yo-DDV/Towk/issues) for
reproducible bugs. Do not open a public issue for a vulnerability; follow the
[security policy](https://github.com/Yo-DDV/Towk/blob/main/SECURITY.md) to
report it privately.
