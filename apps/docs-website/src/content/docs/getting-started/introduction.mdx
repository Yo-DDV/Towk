## What Towk is

Towk is a self-hosted communication workspace for teams and communities. Each
deployment gives one organization its own accounts, rooms, messages, files,
threads, notifications, roles, search, and optional voice and video calls.

Towk is designed to stay compact. A single server binary can serve the web app,
API, realtime connections, and an embedded NATS data store. Larger deployments
can use external NATS, S3-compatible object storage, multiple Towk replicas, and
LiveKit for calls.

The web client is an installable Progressive Web App on desktop and mobile. It
connects directly to the servers you add; there is no central Towk account or
mandatory hosted service.

## What Towk is not

Towk is not a federated protocol. Servers do not exchange community data, and
each deployment remains its own administrative and data-protection boundary.

Towk does not currently offer a hosted edition, paid support plan, or official
mobile-store application. The public repository and its issue tracker are the
source of truth for supported releases, known limitations, and planned work.

## Project status

Towk is pre-1.0 software under active development. Before using it for important
communications:

- pin the exact image digest or source commit you deploy;
- keep regular backups and test restoring them;
- review release notes and configuration changes before upgrading;
- expect public API changes while versions remain in the `0.x` series;
- test browser, push-notification, and call behavior on the devices your users
  rely on.

The [roadmap](https://github.com/Yo-DDV/Towk/blob/main/ROADMAP.md) describes the
current product direction. Reproducible problems and focused usage or
self-hosting questions belong in
[GitHub Issues](https://github.com/Yo-DDV/Towk/issues).

Continue with the [quick start](/getting-started/quick-start/) when you are ready
to run a local instance.
